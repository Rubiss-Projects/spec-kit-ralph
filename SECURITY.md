# Security Policy

## Supported Versions

Security fixes are applied to the latest released version of Ralph Loop. Before reporting a vulnerability, confirm whether it is reproducible with the latest release when it is safe to do so.

## Reporting a Vulnerability

Do not open a public issue for suspected command injection, credential exposure, arbitrary code execution, permission bypass, or another security-sensitive problem.

Use [GitHub's private vulnerability reporting](https://github.com/Rubiss-Projects/spec-kit-ralph/security/advisories/new) instead. Include the affected version, environment, reproduction steps, impact, and any suggested mitigation. Remove real credentials and private repository data from all examples.

Maintainers will acknowledge the report through the private advisory, investigate it, and coordinate disclosure and remediation there. Please avoid public disclosure until a fix or mitigation is available.
